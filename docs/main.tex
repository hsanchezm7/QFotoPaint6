\section{Introducción}

  \textbf{QFotoPaint} es una aplicación de escritorio para el procesamiento de imágenes y vídeo desarrollada en C++, utilizando el \textit{framework} Qt y la librería OpenCV. \\

  Inspirada en herramientas reales de edición como \textit{GIMP} o
  \textit{Photoshop}, QFotoPaint permite al usuario abrir, visualizar y editar
  múltiples imágenes simultáneamente, aplicando diferentes filtros,
  transformaciones y efectos de procesamiento digital de imagen de forma sencilla
  e intuitiva.

\section{Requisitos}

  \noindent QFotoPaint requiere tener instalado un compilador compatible con el estándar C++17 o superior. \\

  Es necesario disponer del entorno de desarrollo Qt Creator, junto con el
  framework Qt y la librería OpenCV, correctamente configurados para compilar y
  ejecutar la aplicación. \\

  \subsection{Nueva imagen desde el portapapeles}

      \noindent
      Esta es una de las operaciones adicionales más sencillas. No es necesario manejar el API del portapapeles de Windows a bajo nivel, pues el
      framework de Qt ofrece una clase ``\textit{wrapper}'' que permite acceder al portapapeles de forma sencilla y ofrece los métodos necesarios
      para la funcionalidad que se desea implementar. \cite{qt_qclipboard} \\

      Por simplicidad, se ha optado por no crear una nueva clase ni un nuevo
      formulario, simplemente definimos una nueva \texttt{QAction} en el menú
      ``Archivo'' y accedemos al portapapeles de la siguiente forma:

      \begin{lstlisting}[style=CppQtStyle, caption={Acceso al portapapeles en Qt.}]
	// mainwindow.cpp
	void MainWindow::on_actionNueva_desde_portapapeles_triggered() {
		const QClipboard *clipboard = QGuiApplication::clipboard();
		const QMimeData *mimeData = clipboard->mimeData();

		if (mimeData->hasImage()) {
			QImage img = qvariant_cast<QImage>(mimeData->imageData());

			if (!img.isNull()) {
				Mat mat = Mat(img.height(), img.width(), CV_8UC4, img.bits());
				cvtColor(mat, mat, COLOR_RGBA2RGB);

				int pl = comprobar_primera_libre();
				if (pl != -1)
				crear_nueva(pl, mat);
			}
		} else {
			QMessageBox::warning(this, "Portapapeles vacío",
			"No se encontró ninguna imagen en el portapapeles.");
		}
	}
\end{lstlisting}

      \noindent
      La aplicación lanzará un diálogo de advertencia si el portapapeles esta vacío o no
      se ha encontrado una imagen en la última entrada del historial.

  \subsection{Copiar al portapapeles}

      \noindent
      Siguiendo la dinámica de la anterior característica relacionada con el
      portapapeles, esta funcionalidad permite al usuario copiar la imagen de la
      ventana seleccionada al portapapeles. Esta operación no necesita la creación de
      una nueva clase ni de un nuevo formulario, y se puede implementar en el propio
      código de \texttt{mainwindow.cpp}.

      \begin{lstlisting}[style=CppQtStyle, caption={Acceso al portapapeles en Qt.}]
    // mainwindow.cpp
    void MainWindow::on_actionCopiar_al_portapapeles_triggered() {
        int f_act = foto_activa();

        if (f_act == -1)
            return;

        ventana &v = foto[f_act];

        Mat roi = v.img(v.roi).clone();

        Mat roiRGB;
        cvtColor(roi, roiRGB, COLOR_BGR2RGB);

        QImage img = QImage((const uchar*)roiRGB.data, roiRGB.cols, roiRGB.rows, roiRGB.step, QImage::Format_RGB888).copy();

        QClipboard *clipboard = QGuiApplication::clipboard();
        clipboard->setImage(img);
    }
\end{lstlisting}

      \noindent
      Realizamos una conversión del modo BGR de OpenCV al modo de color RGB. Se ha intentado convertir al modo de color RGBA pero la imagen contenía errores a la hora de recuperarla. Para actualizar el portapapeles, se hace uso de la operación \texttt{setImage()} proporcionada por la clase \texttt{QClipboard} de Qt. \cite{opencv_histogram_equalization_tutorial}

  \subsection{Deshacer última acción}

      \subsubsection{Introducción}

          \noindent
          Los humanos cometemos numerosos errores y, como cualquier programa de usuario, QFotoPaint6 debería permitir al usuario deshacer los últimos cambios. Esta funcionalidad ha sido una de las más difíciles de implementar, ya que está diseñada de forma que se pueda revertir cualquier cambio cometido en la imagen, ya sea con un formulario o con una herramienta de edición. \\

          Lo primero que nos viene a la mente es definir, para cada ventana (imagen)
          abierta, una lista con el historial de acciones realizadas, para poder navegar
          hacia una de ellas. Sin embargo, en vez de guardar la acción realizada, se
          guarda el ``estado'' de la imagen. Para llevarlo a cabo, será necesario
          modificar la estructura \texttt{ventana}, añadiendo un tipo de dato adecuado
          para soportar las operaciones.

          \begin{figure}[H]
              \centering
              \includegraphics[width=0.6\textwidth]{figures/ventana_undo}
              \caption{Actualización de la estructura ventana con un vector \texttt{img\_hist}.}
              \label{fig:ventana-undo}
          \end{figure}

      \subsubsection{Implementación}

          Para implementarlo en código, modificamos en \texttt{imagenes.h}, añadiendo el
          nuevo vector dinámico que actuará como pila, junto con una constante que
          limitará el tamaño del historial:

          \begin{lstlisting}[style=CppQtStyle, caption={Modificación del \texttt{struct ventana}.}]
	// imagenes.h
	#define MAX_HIST_IMG 20 	// Tamaño del historial para deshacer
	 ...
	struct ventana {
		bool usada;             // Indica si se está usando actualmente
		bool modificada;        // Ha sido modificada después de abrirla o crearla
		string nombre;          // Nombre de la ventana codificada en UTF8
		string nombref;         // Nombre del fichero de imagen codificado en Latin1
		int orden;              // Número de orden entre las ventanas abiertas
		Rect roi;               // Región de interés seleccionada
		Mat img;                // Imagen almacenada en esta posición
		vector<Mat> img_hist;   // Historial de imágenes
	};
\end{lstlisting}

          \clearpage

          Es importante definir métodos que permitan implementar las acciones que
          deseamos soportar. Para ello, se definen los métodos
          \texttt{guardar\_estado()}:

          \begin{lstlisting}[style=CppQtStyle]
	// imagenes.cpp
	void guardar_estado (int nfoto) {
		ventana& v = foto[nfoto];

		v.img_hist.push_back(v.img.clone());  // guardar estado en la pila de deshacer

		if (v.img_hist.size() > MAX_HIST_IMG)	// comprobar tamaño máximo de pila
			v.img_hist.erase(v.img_hist.begin());

		v.modificada = true;
	}
\end{lstlisting}

          \noindent
          y \texttt{deshacer()}:

          \begin{lstlisting}[style=CppQtStyle]
	// imagenes.cpp
	void deshacer (int nfoto) {
		ventana& v = foto[nfoto];
		 ...
		// recuperar y eliminar último estado guardado
		v.img = v.img_hist.back().clone();
		v.img_hist.pop_back();

		imshow(v.nombre, v.img);
	}
\end{lstlisting}

          Para cada acción del menú, el paso a seguir es el mismo para una: guardar el
          estado de la imagen en cada cambio de la ventana. Para ello, añadimos una
          llamada a \texttt{guardar\_estado()} en cada condicional \texttt{if (guardar)}.
          Un ejemplo es el siguiente:

          \begin{lstlisting}[style=CppQtStyle]
	void ver_brillo_contraste_gamma (int nfoto, double suma, double prod, double gamma, bool guardar) {
		 ...
		if (guardar) {
			guardar_estado(nfoto);		// instrucción añadida
			img.copyTo(foto[nfoto].img);
			foto[nfoto].modificada= true;
		}
	}
\end{lstlisting}

          \clearpage

          Sin embargo, si queremos que también funcione para las herramientas (punto,
          línea, arcoíris), al funcionar con eventos, hay que controlar el inicio y fin
          de ``dibujado''. Esto se logra con una variable \texttt{dibujando\_activo} que
          se actualiza en el \textit{callback}, según los eventos
          \texttt{EVENT\_LBUTTONDOWN} y \texttt{EVENT\_LBUTTONUP}.

          \begin{lstlisting}[style=CppQtStyle]
	void callback (int event, int x, int y, int flags, void *_nfoto) {
		 ...
		// 1.5. Se inicia la pulsación del ratón
		if (event==EVENT_LBUTTONDOWN) {
			downx= x;
			downy= y;

			if (!dibujando_activo) {	// marcar inicio trazo
				guardar_estado(factual);
				dibujando_activo = true;
			}
		}

		if (event==EVENT_LBUTTONUP) {		// finalizar trazo
			if (dibujando_activo)
				dibujando_activo = false;
		}
	}
\end{lstlisting}

          \noindent
          De esta forma, cuando el usuario usa, por ejemplo, la herramienta ``Punto'' y mantiene presionado el botón del ratón, solo se guardará un estado de la imagen: el previo a realizar la acción. Además, se ha asignado al QAction correspondiente el atajo de teclado \texttt{CTRL+Z}, aunque no funciona cuando el foco está en una de las ventanas de OpenCV.

  \subsection{Ecualización del histograma}

      \subsubsection{Introducción}

          \noindent
          La ecualización de histogramas es un método utilizado para mejorar el contraste de una imagen, al expandir las zonas que concentran mucho contraste. Su objetivo principal es transformar una imagen de entrada en otra imagen cuyo histograma tiene una distribución uniforme, aunque luego veremos que esto no es exactamente así, pues los histogramas no son completamente planos. \cite{opencv_histogram_equalization_tutorial} \\

          La ecualización de histogramas no funciona igual en imágenes con un solo canal
          (imágenes de grises) que en imágenes a color. Para entender su funcionamiento,
          consideramos una imagen de grises \( X \), siendo \( n_i \) el número de
          ocurrencias para el nivel de grises \( i \). La probabilidad por tanto de que
          un píxel aleatorio tenga el valor de \( i \) es:

          \[ p_X (i) = \frac{n_i}{n}, \quad 0 \leq i < L \]

          \noindent
          siendo \( L \) el número total de niveles de grises en la imagen. \( p_X (i)\) es el valor del histograma para \( i \). \\

          \noindent
          Definimos también la distribución acumulada de píxeles en \( X \) para el valor \( i \).

          \[ \text{cdf}_X(i) = \sum_{j=0}^{i} p_X(j) \]

          Para que esta función pueda emplearse como transformación de intensidades, debe
          normalizarse de manera que su valor máximo corresponda al nivel máximo de
          intensidad permitido (OpenCV usa 255). Es aquí donde empleamos el método que
          nos proporciona la librería de OpenCV \texttt{cv::equalizeHist}.
          \cite{opencv_equalizeHist_function}

          \begin{mainbox}{\large equalizeHist()}
              \textbf{void cv::equalizeHist (InputArray src, OutputArray dst)}

              \vspace{2ex}

              Ecualiza el histograma de una imagen en escala de grises. \\ La función
              ecualiza el histograma de la imagen de entrada utilizando el siguiente
              algoritmo:

              \begin{itemize}
                  \item Calcular el histograma \(H\) para \texttt{src}.
                  \item Normalizar el histograma de manera que la suma de los bins sea 255.
                  \item Calcular el histograma acumulado:

                        \[ H'_i = \sum_{0 \le j < i} H(j) \]

                  \item Transformar la imagen utilizando \( H' \) como tabla de consulta:

                        \[ \text{dst}(x,y) = H'(\text{src}(x,y)) \]
              \end{itemize}

              El algoritmo normaliza el brillo y aumenta el contraste de la imagen.
          \end{mainbox}

          \clearpage

      \subsubsection{Implementación}

          \noindent
          El desarrollo de la ecualización del histograma está fuertemente basada en la del ajuste lineal
          realizado en clase. Según la documentación, la funcionalidad debe permitir al usuario dos modos:

          \begin{itemize}
              \item \textbf{Conjunto:} La operación se aplica de manera conjunta sobre un grupo completo de canales. Concretamente, se realizará sobre un conjunto específico (valor en HSV y luminancia en YCrCb). Es necesario convertir a uno de estos modos ya que, como se especifica en la teoría, pueden aparecer resultados erróneos si usamos el canal RGB, el cual mezcla el color con el brillo. \cite{wikipedia_hist_eq}

              \item \textbf{Independiente:} Si la imagen de entrada consta de varios canales, cada uno se procesa por separado sin tener en cuenta a los demás.
          \end{itemize}

          \noindent
          Para implementar correctamente la funcionalidad, se ha optado por crear una nueva clase junto con un formulario, \texttt{EcualizacionHistograma}. El método implementado en \texttt{imagenes.cpp} es el siguiente:

          \begin{lstlisting}[style=CppQtStyle, caption={Implementación de la ecualización del histograma}, label={lst:ecualhist}]
	// imagenes.cpp
	void ecualizacion_hist(int nfoto, int modo, bool guardar) {
	Mat img = foto[nfoto].img;
	Mat imres;

  \subsection{Incremento del paso en arcoíris}

  \subsection{Introducción}

      \noindent
      Como propuesta del profesor, se ha diseñado una solución para que el usuario sea capaz de modificar el ``\textit{paso}'' con
      el que se cambia de color del pincel en la herramienta ``Arcoíris''. Esta característica permitirá al usuario un mayor control
      sobre apariencia del efecto. \\

      El ``paso'' se refiere a la velocidad de avance que en la secuencia de colores
      del espectro cada vez que se dibuja un píxel. Un paso pequeño provoca que los
      colores cambien lentamente, generando transiciones suaves y graduales entre
      tonos consecutivos. Por el contrario, un paso mayor acelera el cambio de color,
      produciendo un efecto más vivo. \\

  \subsection{Implementación}

      \texttt{QWidget} en \texttt{mainwindow.ui}. Este elemento estará por defecto
      invisible, y su visibilidad cambiará al hacer \textit{click} sobre el botón de
      la herramienta. \\

      él. Se ha tenido que recurrir a un \textit{listener} de eventos de

  \subsection{Rehacer última acción}

      \subsubsection{Introducción}

          \noindent
          Con el fin de completar la funcionalidad de deshacer las acciones realizadas, y con la base ya implementada para soportarla, se ha optado por ofrecer al usuario una forma de recuperar los cambios ``deshechos''. Esta operación es conocida con el nombre de rehacer o \textit{redo} en inglés. La idea es la misma que para la operación de ``deshacer'': modificar la estructura \texttt{ventana} con un nuevo tipo de dato que actúe exactamente igual que la pila implementada anteriormente en la figura~\ref{fig:ventana-undo}. La forma es la siguiente:

          \begin{figure}[H]
              \centering
              \includegraphics[width=0.6\textwidth]{figures/ventana_redo}
              \caption{Actualización de la estructura ventana con un vector \texttt{undo\_hist}.}
              \label{fig:ventana-redo}
          \end{figure}

      \subsubsection{Implementación}

          \noindent
          A nivel de código:

          \begin{lstlisting}[style=CppQtStyle, caption={Modificación del \texttt{struct ventana}.}]
	// imagenes.h
	#define MAX_HIST_IMG 20 	// Tamaño del historial para deshacer/rehacer
	 ...
	struct ventana {
		bool usada;             // Indica si se está usando actualmente
		bool modificada;        // Ha sido modificada después de abrirla o crearla
		string nombre;          // Nombre de la ventana codificada en UTF8
		string nombref;         // Nombre del fichero de imagen codificado en Latin1
		int orden;              // Número de orden entre las ventanas abiertas
		Rect roi;               // Región de interés seleccionada
		Mat img;                // Imagen almacenada en esta posición
		vector<Mat> img_hist;   // Historial de imágenes
		vector<Mat> undo_hist;  // Historial de deshacer
	};
\end{lstlisting}

          \noindent
          Debemos actualizar la operación deshacer para que, en cada cambio de estado, se guarde la imagen actual. De esta manera, la imagen no se pierde y se guarda en la nueva pila. También será necesario definir una nueva operación \texttt{rehacer()}.

          \begin{lstlisting}[style=CppQtStyle, caption={Modificación del \texttt{struct ventana}.}]
    // imagenes.cpp
    void deshacer (int nfoto) {
        ventana& v = foto[nfoto];
         ...
        // guardar estado para poder ir hacia delante
        v.undo_hist.push_back(v.img.clone());
         ...
        if (v.undo_hist.size() > MAX_HIST_IMG)
            v.undo_hist.erase(v.undo_hist.begin());
         ...
    }

    void rehacer (int nfoto) {
        ventana& v = foto[nfoto];

        if (v.undo_hist.empty()) {
            qDebug() << "No hay más estados para rehacer.";
            return;
        }

        // guardar estado en la pila de deshacer
        v.img_hist.push_back(v.img.clone());

        // recuperar y eliminar último estado de la pila de rehacer
        v.img = v.undo_hist.back().clone();
        v.undo_hist.pop_back();

        imshow(v.nombre, v.img);
    }
\end{lstlisting}

          \noindent
          El atajo de teclado designado para el QAction correspondiente es \texttt{CTRL+Y}, el cual, a diferencia del atajo \texttt{CTRL+Z}, no se superpone con ningún otro y se puede ejecutar con el foco en imágenes de OpenCV.
